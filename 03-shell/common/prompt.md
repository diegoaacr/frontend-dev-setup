# 🎛️ Prompt básico

El prompt debe ser limpio, legible y útil.

No debe agregar ruido visual ni depender de configuraciones complejas para funcionar.

---

# 🎯 Objetivo

Tener una terminal clara y cómoda para el trabajo diario.

---

# 📌 Criterios

El prompt recomendado debe ser:

* simple
* rápido
* fácil de mantener
* consistente entre entornos

---

# ✅ Qué debe mostrar

Como base, el prompt puede mostrar:

* usuario
* máquina
* ruta actual

Opcionalmente:

* rama de Git
* estado del entorno virtual o runtime

---

# ⚠️ Qué evitar

* prompts demasiado recargados
* exceso de colores
* información innecesaria
* dependencias difíciles de mantener

---

# 🧠 Recomendación

Empieza con un prompt simple.

Si más adelante necesitas más información visual, agrégala de forma controlada.
